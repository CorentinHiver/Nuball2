\doxysubsection{Detailed Description}
Class used to read .fast files. 

This class is to be used in combination with the \doxyref{Hit}{p.}{class_hit} struct Here is the minimal code you can use to scan the files \+: \begin{DoxyVerb} Hit hit;
 std::string filename;
 FasterReader reader(&hit, filename);

 while(reader.Read())
 {
   print(hit);
 }
\end{DoxyVerb}
 This method deals with the 5 following members of the \doxyref{Hit}{p.}{class_hit} structure\+:(NOT UP TO DATE !!!) \begin{DoxyVerb}   struct Hit
   {
     unsigned short label;  // The label number of the detector
     Float_t        adc;    // The ADC/QDC value of the hit. For RF, contains the frequency.
     Float_t        qdc2;   // The qdc2 value (usually the longer gate)
     ULong64_t      time;   // High precision time in ps (time tick of 7,8125 ps)
     bool           pileup; // Contains either pileup or saturated bit
   }
\end{DoxyVerb}
 -\/-\/-\/-\/-\/-\/-\/--- Define section -\/-\/-\/-\/-\/-\/-\/---

Some options can be activated at compilation time. You have to \#define them before including this library. e.\+g. \+: \begin{DoxyVerb}   ... other includes
   #define FASTER_GROUP
   #include <FasterReader.hpp>
   rest of the code....
\end{DoxyVerb}


-- QDC1\+MAX

By default the qdc2 is handled. If no detector uses the qdc2 then declare QDC1\+MAX

-- FASTER\+\_\+\+GROUP

If the data is grouped using a hardware trigger. From the user point of view, nothing changes but the execution speed. Simply \#define FASTER\+\_\+\+GROUP and then use the class as usual.

The reading is done in two steps \+: first extracts all the hits of the group and put it in a vector, then each call of \doxyref{Read()}{p.}{class_faster_reader_a2d05907241b80ed637b2ae5a01503756} moves in the group. No extra information is extracted \+: from outside of the class, everything goes as if there was no group.

\begin{DoxyAttention}{Attention}
The pileup bit for CRRC4 is not handled 
\end{DoxyAttention}
