\doxysubsection{Detailed Description}
\subsubsection*{template$<$class THist$>$\newline
class MTTHist$<$ THist $>$}

Multithreading wrapper for all THist spectra of root library. 

\begin{DoxyAuthor}{Author}
{\texttt{ corentin.\+hiver@ijclab.\+in2p3.\+fr}}
\end{DoxyAuthor}
Inspiration \+: {\texttt{ https\+://root.\+cern.\+ch/doc/master/\+TThreaded\+Object\+\_\+8hxx\+\_\+source.\+html\#l00167}}

\begin{DoxyAttention}{Attention}
As for any class deriving from \doxyref{MTObject}{p.}{class_m_t_object}, first initialize the number of threads \+: \begin{DoxyVerb} MTObject::Initialize(nb_threads)
\end{DoxyVerb}

\end{DoxyAttention}
Instantiate this class as follow \+: \begin{DoxyVerb} MTTHist<TH1F> some_TH1F_histo("name", "title:xaxis:yaxis", bins, min, max);
\end{DoxyVerb}
 Or \begin{DoxyVerb} MTTHist<TH1F> some_TH1F_histo;
 some_TH1F_histo.reset("name", "title:xaxis:yaxis", bins, min, max);
\end{DoxyVerb}
 \begin{DoxyAttention}{Attention}
Please do not use the copy constructors and operators, may need proper work to make it safe
\end{DoxyAttention}
In default mode, nb\+\_\+threads histograms are created \begin{DoxyRefDesc}{Test}
\item[\textbf{ Test}]In mono mode (MTTHIST\+\_\+\+MONO), only one histogram is created UNDER DEVELOPPEMENT\end{DoxyRefDesc}


To fill the histogram from threads \+: \begin{DoxyVerb}     some_TH1F_histo.Fill()
\end{DoxyVerb}
 Once the histogram have been filled, two options \+:


\begin{DoxyItemize}
\item Either write it down directly \+: \begin{DoxyVerb}  // Open a TFile
  some_TH1F_histo.Write()
  // Write and close the TFile
\end{DoxyVerb}

\item Or you can merge the histograms \+: \begin{DoxyVerb} some_TH1F_histo.Merge();
\end{DoxyVerb}
 You can then address the merged histogram using -\/$>$ \+: \begin{DoxyVerb}some_TH1F_histo->Integral();
\end{DoxyVerb}
 Special case \+: you can also instanciate and write down one histogram per thread.
\end{DoxyItemize}

Example \+: void single\+\_\+thread(\+MTTHist \& histo) \{ histo.\+reset(\char`\"{}name\char`\"{},\char`\"{}title\char`\"{}, args...); ... Fill the histo histo.\+Write(); \}

int main() \{ MTTHist$<$\+TH1$>$ on\+\_\+histo\+\_\+per\+\_\+thread; std\+::parallelise\+\_\+function(single\+\_\+thread, on\+\_\+histo\+\_\+per\+\_\+thread); return 1; \}

In this case each histogram is kept separated, no merge occurs. Use this if you want to make the benefits of the user defined method of this class or for consistency with the rest of the code. It is equivalent to declare the histogram directly inside the funcion
