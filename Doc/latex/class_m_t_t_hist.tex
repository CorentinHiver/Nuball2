\doxysubsection{Detailed Description}
\subsubsection*{template$<$class THist$>$\newline
class MTTHist$<$ THist $>$}

Multithreading wrapper for all THist spectra of root library. 

\begin{DoxyAuthor}{Author}
{\texttt{ corentin.\+hiver@free.\+fr}}
\end{DoxyAuthor}
Inspiration \+: {\texttt{ https\+://root.\+cern.\+ch/doc/master/\+TThreaded\+Object\+\_\+8hxx\+\_\+source.\+html\#l00167}}

\begin{DoxyAttention}{Attention}
As for any class using my multithreading, Initialise the number of threads BEFORE instantiating any object \+: \begin{DoxyVerb} MTObject::Initialise(nb_threads)
\end{DoxyVerb}

\end{DoxyAttention}
Instantiation \+: \begin{DoxyVerb} MTTHist<TH1F> some_TH1F_histo(<normal arguments for TH1F objects>);
\end{DoxyVerb}
 Or \begin{DoxyVerb} MTTHist<TH1F> some_TH1F_histo;
 some_TH1F_histo.reset("name", "title:xaxis:yaxis", bins, min, max);
\end{DoxyVerb}
 In default mode, [nb\+\_\+threads] sub\+\_\+histograms are created \begin{DoxyRefDesc}{Test}
\item[\textbf{ Test}]In mono mode (MTTHIST\+\_\+\+MONO), only one histogram is created UNDER DEVELOPPEMENT\end{DoxyRefDesc}


To fill the histogram from within one thread \+: \begin{DoxyVerb}     some_TH1F_histo[MTObject::getThreadIndex()]->Fill(<normal arguments for TH1*::Fill()>)
\end{DoxyVerb}
 Or in a more concise way (preferred) \+: \begin{DoxyVerb}     some_TH1F_histo.Fill(<normal arguments for TH1*::Fill()>)
\end{DoxyVerb}
 Once the histograms have been filled, two options \+:

Either write it down directly \+: \begin{DoxyVerb}    // Open a TFile and cd() inside
    some_TH1F_histo.Write(); //Merges the sub_histograms then write it if it has counts
    // Write and close the TFile
\end{DoxyVerb}
 Or you can merge the histograms \+: \begin{DoxyVerb}   some_TH1F_histo.Merge();
\end{DoxyVerb}
 You can then address the merged histogram using -\/$>$ \+: \begin{DoxyVerb}some_TH1F_histo->Integral();
some_TH1F_histo->GetEntries();
some_TH1F_histo->Write();
...
\end{DoxyVerb}
