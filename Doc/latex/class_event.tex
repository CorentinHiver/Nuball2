\doxysubsection{Detailed Description}
\doxyref{Event}{p.}{class_event} used for reading and writting event, event building and trigger. 

An \doxyref{Event}{p.}{class_event} is in principle a collection of Hits. However, in order to be an efficient interface with ROOT TTree, in practice it consist of a collections of arrays that stores values of a specific branch.

The following are public members (i.\+e. you can call it directly with \doxyref{Event}{p.}{class_event}\+:\+:[array\+\_\+name]) \begin{DoxyVerb}   int mult = 0;                  Number of hits currently stored in the event.
   Timestamp stamp = 0ull;         Absolute timestamp of the whole event
   Label   labels  [255] = {0};    Labels of the hits
   Time    times   [255] = {0};    Time in ps (Long64_t) relative to the first hit
   Time_ns time2s  [255] = {0};    Time in ns (float) relative either to the first hit or to the pulse
   ADC     adcs    [255] = {0};    Uncalibrated ADC value of the energy
   NRJ     nrjs    [255] = {0};    Calibrated (or simply gain matched) energy value in keV
   ADC     qdc2s   [255] = {0};    Uncalibrated QDC value of the energy measured in the second time gate for QDC2 channels
   NRJ     nrj2s   [255] = {0};    Calibrated (or simply gain matched) energy value in keV. By default, the energy calibration is the same as the nrj so that the ratio remains the same
   Pileup  pileups [255] = {0};    Pileup or saturation bit
\end{DoxyVerb}
 Format of a branch inside the root tree \+: \begin{DoxyVerb}   "array[mult]/type"
\end{DoxyVerb}
 Only the two first members are not arrays because they are true for the entire event.

To read any Nuball2-\/like TTree \+: \begin{DoxyVerb}   // Loads the TTree or TChain ...
   Event event;
   event.reading(ttree);
\end{DoxyVerb}
 If you are only interested in a few branches, you can choose them by adding an option \+: \begin{DoxyVerb}   event.reading(ttree, "lst"); // Reads only the multiplicity, label, timestamp and relatative time.
\end{DoxyVerb}
 Note the multiplicity will always be activated, because it is mandatory to read the root file.

A list of all the options available so far \+: \begin{DoxyVerb}   l : label  label                 ushort
   s : stamp  absolute timestamp ps ULong64_t
   t : time   relative timestamp ps Long64_t
   T : time2  relative timestamp ns float
   e : adc    energy in ADC         int
   E : nrj    energy in keV         float
   q : qdc2   energy qdc2 in ADC    float
   Q : nrj2   energy qdc2 in keV    float
   p : pileup pilepup               bool
\end{DoxyVerb}
 You can access the readed branches via the \doxyref{Event\+::read}{p.}{class_event_a9c01144b3f06d4cc5234ae3736fec9d2} member (see the Read\+IO struct definition)

You can use this class to write in another root tree. To do so, use the \doxyref{Event\+::writting}{p.}{class_event_a0b93fcd8063a3c02edede96bb53b136d} method \+: \begin{DoxyVerb}   event.writting(outTree, "lstEQ"); Will write the multiplicity, timestamp, relative time and calibrated energy and QDC2
\end{DoxyVerb}
 You can as well access the written branches via the \doxyref{Event\+::write}{p.}{class_event_a4db4697dc48d987a650c89b92404f51d} member (see the Write\+IO struct definition)

Interface with \doxyref{Hit}{p.}{class_hit} class \+:

You can add Hits to an event using the \doxyref{Event\+::push\+\_\+back}{p.}{class_event_a65f680e6b4c74419f4aa92e6e772efb0} or \doxyref{Event\+::push\+\_\+front}{p.}{class_event_a78b1cb630e7720f205a9c00e2c111102} methods \+: \begin{DoxyVerb}   event.push_back(hit);

   event.push_front(hit);
\end{DoxyVerb}
 When the event is complete (e.\+g. in an event builder), you can for instance write it down or analyse it, then call \doxyref{Event\+::clear()}{p.}{class_event_a0862af1471604bf30a4f50a5e028637a} to empty it.
